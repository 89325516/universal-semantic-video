# Risk Register

## English

| Risk | Impact | Current Control | Next Review |
| --- | --- | --- | --- |
| The project is mistaken for a codec or container replacement. | Misaligned adoption and standards friction. | README, PRD, TRD, and standards docs state non-goals. | Before any embedding work |
| Private source notes or media enter the public repository. | Privacy and trust failure. | `.gitignore`, contribution rules, release scans. | Every release |
| Schema grows faster than implementer feedback. | Poor interoperability. | Roadmap requires feedback before 1.0 profile freeze. | 0.2 |
| Rights and provenance fields are treated as enforcement. | Legal and safety ambiguity. | Docs define them as declarative metadata and references. | 0.2 |
| Container embedding starts before mappings are reviewed. | Format lock-in or incompatible hacks. | Embedding is deferred to 0.4 experiments. | 0.4 |
